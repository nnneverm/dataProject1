点击选择 .zip 文件，或拖拽至此处
包内包含 data/ + models/（可选）+ code/（可选）+ README.md（首版必须），将自动推送至 LakeFS
🗄 每次发版生成一条不可变记录，自动写入 DVC 或 LakeFS，支持随时回溯任意历史版本
💡 在训练服务器上直接推送 数据目录 + 模型目录，无需打包。推荐用环境变量固化凭据和仓库路径，之后每次上传只需两行命令。
1 写入环境变量（首次，加到 ~/.bashrc 后 source 生效）
# LakeFS 服务地址（平台统一，无需修改）
export LAKECTL_SERVER_ENDPOINT_URL=https://lakefs.cvforce.example.com
# 个人访问密钥（平台「个人设置 → Access Keys」获取）
export LAKECTL_CREDENTIALS_ACCESS_KEY_ID=<your-access-key>
export LAKECTL_CREDENTIALS_SECRET_ACCESS_KEY=<your-secret-key>
# 安装 lakectl（首次）
curl -L https://lakefs.cvforce.example.com/dl/lakectl -o /usr/local/bin/lakectl && chmod +x /usr/local/bin/lakectl
2 设置当前项目仓库路径（每个项目设置一次，复制下方命令即可）
# Project ID 已自动填入，直接执行即可
export REPO=lakefs://svc-xxx/main
3 上传目录 & 提交版本（每次发版执行）
# 上传数据目录（-r 递归，只传有变化的文件）
lakectl fs upload -r $REPO/data/ ./data/
# 上传模型目录（如有）
lakectl fs upload -r $REPO/models/ ./models/
# 上传代码目录（训练 / 推理脚本）
lakectl fs upload -r $REPO/code/ ./code/
# 提交版本快照，平台自动生成版本历史记录
lakectl commit $REPO -m "v1.2.0: 新增夜间场景样本500条"
🗄 lakectl fs upload 只传差异对象，未修改的文件不会重复占用存储。lakectl commit 完成后平台 Webhook 自动入库，记录出现在「版本历史」列表。
